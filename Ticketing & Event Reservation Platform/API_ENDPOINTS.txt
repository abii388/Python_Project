Concert Booking API Endpoints

Base URL: http://<host>:<port>/

Common header for authenticated requests:
Authorization: Token <token>

1) POST /signup
Description: Create a new user account.
Request body:
{
  "username": "john",
  "email": "john@example.com",
  "password": "123456",
  "confirm_password": "123456",
  "ph_no": "9876543210"
}

2) POST /login
Description: Login and get an authentication token.
Request body:
{
  "username": "john",
  "password": "123456"
}

3) POST /logout
Description: Logout the current authenticated user.
No JSON body required.

4) POST /createconcert
Description: Create a new concert (admin only).
Request body (JSON):
{
  "concert_name": "Rock Night",
  "concert_date": "10-07-2026",
  "concert_time": "19:00",
  "concert_location": "Delhi",
  "prize": 500,
  "available_tickets": "100",
  "status": "Enable"
}

If you add an image field, send it as multipart/form-data instead of JSON:
- Field name: image
- Example values:
  - concert_name: Rock Night
  - concert_date: 10-07-2026
  - concert_time: 19:00
  - concert_location: Delhi
  - prize: 500
  - available_tickets: 100
  - status: Enable
  - image: [select image file]

5) GET /retrieve_concert
Description: List all concerts.
No JSON body required.

6) GET /concert_details/<id>
Description: Get one concert by id.
No JSON body required.

7) PUT /concert_details/<id>
Description: Update a concert (admin only).
Request body:
{
  "concert_name": "Rock Night Updated",
  "concert_date": "12-07-2026",
  "concert_time": "20:00",
  "concert_location": "Mumbai",
  "prize": 600,
  "available_tickets": "80",
  "status": "Enable"
}

8) DELETE /concert_details/<id>
Description: Delete a concert (admin only).
No JSON body required.

9) GET /search?search=<value>
Description: Search concerts by name, location, or date.
Example:
GET /search?search=Delhi
No JSON body required.

10) GET /booked_concert_details/<id>
Description: View booked tickets for a concert (admin only).
No JSON body required.

11) GET /status_update/<id>
Description: Toggle the concert status between Enable and Disable (admin only).
No JSON body required.

12) GET /user_details
Description: View all registered users (admin only).
No JSON body required.

13) GET /user_view/<id>
Description: Get one user by id (admin only).
No JSON body required.

14) PUT /user_view/<id>
Description: Update a user (admin only).
Request body:
{
  "username": "john_updated",
  "email": "john_updated@example.com",
  "ph_no": "9999999999"
}

15) DELETE /user_view/<id>
Description: Delete a user (admin only).
No JSON body required.

16) GET /book_concert/<c_id>
Description: Get the ticket price of a concert.
No JSON body required.

17) POST /confirm_concert/<c_id>
Description: Book tickets for a concert.
Request body:
{
  "ticket_no": 2
}

18) GET /user_history
Description: View the current user's booking history.
No JSON body required.

19) DELETE /delete_history/<c_id>
Description: Delete a booking from the current user's history.
No JSON body required.

20) GET /pdf_generate/<id>
Description: Download a booking PDF.
No JSON body required.

Notes:
- Admin-only endpoints require a user with user_role = admin.
- Date format used by the app is DD-MM-YYYY.
- The booking endpoint allows a maximum of 3 tickets per request.
